## 🎯 概要

**インターナルセールス自動化プラン**は、インターナルセールス(IS)チームの反復的な管理業務を自動化するための包括的なソリューションです。このプロジェクトは、営業チームが最も重要なこと、つまり販売と顧客関係の構築に集中できるように、時間のかかる手作業のワークフローに対処します。

### 解決する課題

インターナルセールスチームは通常、**業務時間の60〜70%**を管理業務に費やしています:

- ✉️ 会議リマインダーの手動送信
- 📝 通話記録システムからのトランスクリプトのコピー
- 📄 通話サマリードキュメントの作成
- 📅 フィールドセールス担当者のカレンダー更新
- 🔄 複数のシステムへのデータ入力

**この自動化プランはこれらの手作業を排除し**、営業チームが収益を生み出す活動に集中できるようにします。

---

## 🚀 この自動化プランの機能

このリポジトリは、2つの実装オプションを備えた完全な自動化システムを提供します:

### 1. **会議リマインダー自動化**
会議参加者にプロフェッショナルなリマインダーメールを自動送信:
- 会議の**24時間前**
- 会議の**1時間前**
- HubSpotで会議がスケジュールされたときに自動トリガー
- 会議の詳細、参加リンク、アジェンダを含む

### 2. **通話サマリー自動化**
インテリジェントな通話サマリーを自動生成:
- Miitel(通話録音プラットフォーム)から通話トランスクリプトを取得
- AI (OpenAI GPT-4)を使用して構造化されたサマリーを生成
- 通話ログに自動的にサマリーを投稿
- **通話あたり10〜15分**を節約

### 3. **カレンダー同期**
フィールドセールス担当者に情報を提供:
- フィールドセールス担当者のGoogleカレンダーを自動更新
- 会議の詳細と顧客情報を同期
- スケジュールの競合を防止
- チーム間でリアルタイムの可視性を確保

## ✨ 主な機能

### 🔔 スマート会議リマインダー
- **自動検出**: HubSpotで会議が作成されたときにトリガー
- **設定可能なタイミング**: カスタムリマインダー間隔の設定(デフォルト: 24時間と1時間)
- **プロフェッショナルなテンプレート**: カスタマイズ可能なメールテンプレート
- **複数参加者サポート**: すべての出席者に自動送信
- **キャンセル処理**: 再スケジュールまたはキャンセルされた会議を管理

### 🤖 AI搭載通話サマリー
- **自動トランスクリプト取得**: Miitel APIとの直接統合
- **インテリジェントな要約**: コンテキストを考慮したサマリーにGPT-4を使用
- **カスタムプロンプト**: 異なる通話タイプ(発見、デモ、フォローアップ)に設定可能
- **マルチシステム更新**: HubSpotと通話ログを自動更新
- **品質保証**: 投稿前のオプションの手動レビュー

### 📊 カレンダー統合
- **リアルタイム同期**: 会議作成から5分以内に更新
- **スマートマッピング**: ISチームメンバー → FS担当者のマッピング
- **競合検出**: スケジュールの競合を識別してフラグ付け
- **詳細情報**: 顧客コンテキストと会議メモを含む

### 🛠️ 管理機能
- **ダッシュボード**: 自動化のパフォーマンスを監視
- **アクティビティログ**: すべての自動化アクションを追跡
- **エラー通知**: 失敗の即座のアラート
- **設定パネル**: 簡単な設定管理
- **分析**: 成功率、節約時間、パフォーマンスメトリクス

## 🔍 課題と解決策

### 現在の状態(自動化前)

**インターナルセールスチームメンバーの日常業務:**

```
朝:
- HubSpotで会議を確認 → 5分
- リマインダーメールを手動送信 → 30分
- 前日の通話トランスクリプトをレビュー → 20分

通話ごと(平均1日10件):
- Miitelにアクセスしてトランスクリプトをコピー → 3分
- 通話サマリーテンプレートを開く → 1分
- 手動でサマリーを作成 → 8分
- 通話ログにサマリーを貼り付け → 2分
- FS担当者のカレンダーを更新 → 2分
通話あたり合計: 16分 × 10件 = 160分(2.6時間)

管理業務の合計時間: 約3.5時間/日
```

**課題:**
- ❌ 時間のかかる手動プロセス
- ❌ 人的エラーが発生しやすい
- ❌ サマリーの品質が一貫しない
- ❌ カレンダー更新の遅延
- ❌ リマインダーメールの送信漏れ
- ❌ 反復作業によるチームの燃え尽き

### 将来の状態(自動化後)

**インターナルセールスチームメンバーの日常業務:**

```
朝:
- 自動化ダッシュボードを確認 → 2分
- フラグ付き項目をレビュー(ある場合) → 5分

通話ごと:
- サマリーが自動生成される → 0分
- カレンダーが自動更新される → 0分
- 通話ログが自動更新される → 0分

リマインダー:
- 設定された時刻に自動送信 → 0分

管理業務の合計時間: 約10分/日
```

**メリット:**
- ✅ **時間の節約**: チームメンバー1人あたり1日3時間以上
- ✅ **精度**: AIサマリーで99%以上の精度
- ✅ **一貫性**: 標準化されたプロセスとテンプレート
- ✅ **リアルタイム更新**: システム間での即座の同期
- ✅ **リマインダーの送信漏れゼロ**: 100%信頼性の高い自動化
- ✅ **チームの士気**: 意味のある仕事に集中

## ❓ よくある質問

**Q: プログラミング経験は必要ですか?**
A: いいえ! Zapierアプローチはコーディングゼロです。コードベースアプローチはカスタマイズのためにNode.js/TypeScriptの基本的な知識が必要です。

**Q: セットアップにはどのくらい時間がかかりますか?**
A: Zapierアプローチ: 2〜4週間。コードベースアプローチ: 8〜12週間。クイックスタートは2時間で可能です。

**Q: 通話録音システムがMiitelでない場合はどうなりますか?**
A: アーキテクチャはAPIまたはWebhook機能を持つあらゆるシステムをサポートします。統合レイヤーを調整する必要があります。

**Q: AIサマリープロンプトをカスタマイズできますか?**
A: はい! 両方のアプローチでサマリーテンプレートとプロンプトの完全なカスタマイズが可能です。

**Q: コストはいくらですか?**
A: Zapier: 年間約7万円。コードベース: 年間約14〜30万円(インフラ)。コスト比較は[実装戦略](./IMPLEMENTATION_STRATEGY.md)を参照してください。

**Q: データは安全ですか?**
A: はい。両方のアプローチは暗号化を使用した安全なAPI接続を使用します。コードベースソリューションは完全なデータ制御を提供します。

**Q: 小規模から始めてスケールアップできますか?**
A: もちろんです! 会議リマインダーのみから始めて、準備ができたら通話サマリーとカレンダー同期を追加できます。

**Q: 自動化が失敗した場合はどうなりますか?**
A: 両方のアプローチにエラー処理と通知が含まれています。失敗があればすぐにアラートを受け取ります。
